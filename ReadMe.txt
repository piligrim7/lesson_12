Проект реализует парсер вакансий hh.ru по API c выводом результатов аналитики
Интерфейс представляет собой сайт с тремя страницами:
Главная - страница с краткой информацией
Аналитика по вакансиям - форма ввода запросов
Контакты - страница контактов
Страница аналитики - страница с результатами (недоступна из меню)

Ввод данных запроса производится на странице Аналитика по вакансиям
В первом поле ввода нужно ввести искомую вакансию
Во втором поле вводим количество анализируемых предложений:
    данное число определяет количество первых результатов запроса, которые будут
    использованы для сбора статистики и анализа.
    (Рекомендуется число не более 50, иначе ожидание ответа будет долгим)
В третьем поле вводим количество результатов для вывода информации (размер ТОП листа)

После нажимаем кнопку Запрос данных и ждем результат анализа, который будет открыт
на отдельной странице:
    Количество найденных вакансий
    Средняя заработная плата
    Топ городов с вакансиями
    Топ необходимых навыков по найденным вакансиям

В проекте реализовано сохранение данных по ранее запрашиваемым вакансиям в БД SQLite
с использованием orm sqlalchemy,
соответственно при наличии в выборке с сайта hh.ru сохраненных в БД вакансий
(сопоставление по ID) выполняется извлечение из БД.
Если по ID записи в БД не найдены, делается запрос по данным вакансии с hh.ru.
Реализованный механизм записи/чтения данных из БД позволяет значительно сократить
скорость повторной выборки
При первом запросе создается база данных hh.db в соответствии с классами в model.py
